	There's a bug on x64 systems, where the mms.cfg is expected in ..\System32\Macromed\Flash.
	By default, mms.cfg is put to ..\SysWOW64\Macromed\Flash.
	This problem is not solveable with MSI, because ..\System32 is consequently redirected to ..\SysWOW64
	Therefore we use an xcopy command in a CA to do this.

	Insert into Component table:
	CfgFileDuplicate	{BBA2BDE6-82A7-49EE-8E7A-EADDFE8BC399}	INSTALLDIR	0	(VersionNT64)	

	Insert into CustomAction table:
	CopyCfg2x64	3170	FLASH	xcopy.exe "[FLASH]mms.cfg" "C:\Windows\System32\Macromed\Flash\" /y	

	Insert into Directory table:
	System64Folder	TARGETDIR	.:System32
	MACROMED64	System64Folder	Macromed
	FLASH64	MACROMED64	Flash

	Insert into FeatureComponents table:
	FlashPlayerAXInstaller	CfgFileDuplicate

	Insert into InstallExecuteSequence table:
	CopyCfg2x64	NOT REMOVE	4050
